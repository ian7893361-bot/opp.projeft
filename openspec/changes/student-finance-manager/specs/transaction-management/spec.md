## ADDED Requirements

### Requirement: 使用者可以新增收入記錄
系統 SHALL 允許已登入的使用者新增收入，輸入欄位包含：金額、來源類別（如：打工、獎學金、紅包）、日期、備註。

#### Scenario: 成功新增收入
- **WHEN** 使用者選擇「新增收入」並填寫有效的金額、類別、日期
- **THEN** 系統建立 `Income` 物件，加入該使用者的 `vector<Transaction*>`，並儲存至 `data/txn_<id>.txt`

#### Scenario: 金額無效（非正數）
- **WHEN** 使用者輸入負數或零作為金額
- **THEN** 系統顯示「金額必須大於零」提示，要求重新輸入

### Requirement: 使用者可以新增支出記錄
系統 SHALL 允許使用者新增支出，輸入欄位包含：金額、支出類別（如：餐費、交通、學費）、支出類型（固定/臨時）、日期、備註。

#### Scenario: 成功新增固定支出
- **WHEN** 使用者選擇「新增支出」並選擇固定支出類型（如學費）
- **THEN** 系統建立 `FixedExpense` 物件並儲存

#### Scenario: 成功新增臨時支出
- **WHEN** 使用者選擇「新增支出」並選擇臨時支出類型（如餐費）
- **THEN** 系統建立 `OneTimeExpense` 物件並儲存

### Requirement: 使用者可以查詢交易記錄
系統 SHALL 允許使用者查詢所有交易或依條件篩選（類別、月份）。

#### Scenario: 查詢全部記錄
- **WHEN** 使用者選擇「查看全部記錄」
- **THEN** 系統以時間排序顯示所有交易，每筆格式為：`[日期] [類型] [類別] [金額] [備註]`

#### Scenario: 依月份篩選
- **WHEN** 使用者輸入年月（如 2026-06）
- **THEN** 系統以 `find_if` 篩選出該月份的交易並列出

### Requirement: 使用者可以刪除交易記錄
系統 SHALL 允許使用者依交易 ID 刪除指定的交易記錄。

#### Scenario: 成功刪除
- **WHEN** 使用者輸入存在的交易 ID 並確認刪除
- **THEN** 系統從 `vector<Transaction*>` 移除該記錄，釋放記憶體，並更新檔案

#### Scenario: 交易 ID 不存在
- **WHEN** 使用者輸入不存在的交易 ID
- **THEN** 系統顯示「找不到該記錄」並返回選單，不修改資料
